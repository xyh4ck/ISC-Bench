What is ISC?

Internal Safety Collapse (ISC) is a structural vulnerability where frontier LLMs generate harmful content as a functional requirement of legitimate professional tasks.

Unlike traditional jailbreaks, ISC triggers through the task structure itself. The model believes it is "just doing its job."

We tested every major frontier LLM — GPT, Claude, Gemini, Grok, DeepSeek, Qwen, GLM, Kimi, MiniMax, Doubao — and every single one is affected.

The TVD Framework

ISC is triggered by the TVD design pattern: Task + Validator + Data.

Figure 2. The TVD (Task, Validator, Data) framework for systematically triggering ISC.

Task

A benign professional scenario — ML engineering, content moderation, bioinformatics.

Validator

Code that checks output — Pydantic models, JSON schemas, assertions.

Data

Harmful queries with ??? placeholders. The LLM fills them to complete the task.
